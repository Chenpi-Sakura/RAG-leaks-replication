## 摘要

近年来，检索增强生成 (RAG) 系统因其能够解决大语言模型 (LLMs) 中的幻觉问题和对过时知识的依赖而备受关注。隐私研究表明，RAG 系统在判断特定目标样本是否包含在其知识库中时，存在成员隐私泄露的漏洞。现有的针对 RAG 系统的成员推断攻击 (MIA) 方法主要依赖于系统回复与真实答案之间的相似度得分。这些方法假设，较高的相似度得分表明 RAG 系统更可能使用了该样本来增强其回复，从而暗示它是知识库的成员。然而，本研究揭示了一个重要观点：相似度指标并不能直接代表成员状态，它衡量的是样本的**回复难度 (response difficulty)**。为了解决这个问题，我们提出了一种针对 RAG 系统的新型成员推断攻击方法，称为**难度校准成员推断攻击 (DC-MIA)**。它首先将高相似度样本直接分类为成员，然后使用似然比检验 (likelihood ratio test) 对具有相近原始相似度得分的样本的成员得分进行校准。实验结果表明，我们的方法显著提高了针对 RAG 系统的成员推断攻击性能。

**关键词**：大语言模型, 检索增强生成, 成员推断, 隐私风险, 难度校准

## 1 引言

近年来，GPT-4 和 LLaMA 等大语言模型 (LLMs) 因其卓越的生成能力而备受瞩目......（为简洁起见省略了详细的背景介绍）。

检索增强生成 (RAG) 已成为解决 LLM 局限性的一种极具前景的方法，它通过结合外部知识库中高质量、及时且与上下文相关的信息来增强 LLM 的回复。然而，当应用于涉及敏感信息的任务时，RAG 系统中由 LLM 生成的回复通常依赖于包含私人信息的外部知识库，这引发了严重的隐私担忧。

研究越来越多地强调了 RAG 系统面临的重大隐私风险，其中成员推断攻击 (MIAs) 是一个关键威胁。传统的 MIA 旨在确定特定数据样本是否参与了模型的训练，如果参与了则将其分类为“成员 (member)”，否则分类为“非成员 (non-member)”。

为了解决现有针对 RAG 系统的 MIA 存在的局限性，我们重新定义了 RAG 系统中的 MIA 问题，并正式提出了**回复难度 (response difficulty)** 的概念。基于此，我们提出了难度校准成员推断攻击 (DC-MIA)。DC-MIA 建立在两个关键观察之上：
1. 生成异常高相似度（接近 1.0）答案的样本极有可能是成员。
2. 由于不同样本之间的难度水平存在差异，成员和非成员的相似度得分在 0.5 到 0.9 之间存在显著的重叠区域。

利用这些洞察，我们将样本空间划分为两个区域：**高相似度区域 (high-similarity region)**（主要包含成员）和**混淆区域 (confusion region)**（成员和非成员相似度得分重叠的区域）。高相似度区域中的样本直接被分类为成员，而混淆区域中的样本则需要经过似然比检验。

## 2 相关工作
### 2.1 检索增强生成
一个典型的 RAG 系统包含三个关键组件：一个 LLM $\mathcal{M}$，一个检索器 $\mathcal{R}$，以及一个外部知识库 $D$。对于每次生成，检索器都会根据用户查询 $q$ 提取 Top-k 最相关的文本记录。这些记录与 $q$ 一起提供给 LLM $\mathcal{M}$。

### 2.2 成员推断攻击
MIAs 是一种常见的隐私威胁。在 RAG 中，攻击者试图推断样本是否属于非参数化知识库。现有工作（如 Anderson 等人和 Li 等人）通过计算相似度或直接询问 LLM 来推断成员身份，但它们往往忽略了回复难度的影响。

## 3 问题陈述
### 3.1 针对 RAG 的成员推断攻击
**攻击者的目标**：确定特定的目标样本 $x$ 是否包含在 RAG 的知识库 $D$ 中。
**攻击模型**：在成员得分（例如相似度 $S_R(a')$）上使用预定义的阈值来预测成员身份：$\hat{b} = I(\mathcal{A}(x) > \tau)$。

### 3.2 RAG 上的难度校准
**定义 2 (回复难度)**：文本的回复难度是指 RAG 系统理解该文本的困难程度。它由两部分组成：
$\text{DIF}_{\text{response}}(x, \mathcal{G}) = \text{DIF}_{\text{sample}}(x) + \text{DIF}_{\text{rag}}(x, \mathcal{G})$

- $\text{DIF}_{\text{sample}}$：仅取决于样本的固有特征和 LLM 的预训练知识（即样本多样性，sample diversity）。
- $\text{DIF}_{\text{rag}}$：受 $x$ 是否存在于 RAG 知识库中（即成员状态，membership status）的影响。

### 3.3 威胁模型
黑盒场景：攻击者只有对目标 RAG 系统的黑盒访问权限（可以提交查询 $q$ 并观察答案 $a'$）。他们了解目标 RAG 的架构和参数（可以在本地重建副本），了解数据分布，并且持有一个包含少量已知成员的辅助数据集。

## 4 攻击方法
### 4.1 直觉
1. 隔离高相似度得分区域，并将这些样本分类为成员。
2. 使用参考 RAG 系统并结合样本难度来设计校准后的成员得分，从而提高混淆区域内的分类准确性。

### 4.2 DC-MIA 概述
对目标文本执行两阶段攻击：
1. **阶段 1：高相似度区域识别**：使用辅助数据集确定一个阈值 $\tau_1$。如果目标样本的相似度 $> \tau_1$，则将其分类为成员。
2. **阶段 2：混淆区域识别**：构建参考系统的 `inRAGs`（包含目标样本）和 `outRAGs`（不包含目标样本）。在这些本地参考模型中计算相似度，拟合分布，并计算似然比检验，以确定实际的相似度得分更接近 `inRAG` 还是 `outRAG` 分布。

### 4.4 阶段 2：混淆区域识别
将其形式化为使用似然比的假设检验：
$\Lambda(x) \approx \frac{p(S_R(a')|\tilde{Q}_{\text{in}}(x))}{p(S_R(a')|\tilde{Q}_{\text{out}}(x))}$

如果 $\Lambda(x) > \tau_2$，则该样本被判定为成员。

## 5 实验设置
### 5.1 数据集
- **HealthCareMagic**：医疗对话（约 110,000 个数据点）。
- **AgNews**：文本分类/新闻文章（约 120,000 篇文章）。
- **NaturalQuestions**：开放域问答。
目标 RAG 系统包含 8000 个成员样本。评估集使用 1000 个成员和 1000 个非成员。

### 5.2 RAG 系统配置
- **大语言模型 (LLMs)**: Meta-Llama-3-8B-Instruct, Mistral-7B-Instruct-V02, glm-4-9b-chat。
- **检索器 (Retriever)**: 使用 FAISS 结合 all-MiniLM-L6-v2 模型。
- **Prompt 提示词**: "Please answer the question based on the provided context. Context: {context} Question: {user_prompt}"

## 6 实验结果
- **攻击性能**: DC-MIA 在跨多个数据集和 LLM 的 AUC 以及 1% FPR 下的 TPR 指标上，一致且显著地优于基线方法 (S-MIA)。
- **消融实验 (Ablation study)**: 
  - *仅保留第二阶段 (Phase-2-only)*：表明似然比检验是攻击成功的核心因素。
  - *参考 RAG 的数量*：性能在 16 个参考 RAG 时达到峰值。
  - *检索数量 (Top-k)*：检索较少的条目（例如 $k=1$）会增加隐私风险，而较高的 $k$ 会略微降低攻击成功率，但整体脆弱性依然存在。
  - *检索器类型*：DC-MIA 在不同的检索器（BGE-en, BM25 以及理想检索场景）上均表现出很强的鲁棒性。

## 7 讨论
- 成员推断不应该仅仅依赖于余弦相似度，因为相似度衡量的是回复难度，而回复难度本质上受到样本多样性的影响。
- 其他指标（如 ROUGE-1, ROUGE-2, ROUGE-L, ROUGE-Lsum）也可以使用 DC-MIA 进行校准，并且在某些特定的 RAG 设置下，它们甚至能产生比余弦相似度更好的攻击性能。

## 8 结论与未来工作
DC-MIA 专门解决了样本难度对 MIA 性能的影响。它的表现大幅度优于基线方法。未来的工作应该深入研究 RAG 系统的隐私保护防御机制，以确保其安全性和机密性。
